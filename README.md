# もじレーシング (MojiRacing)

タイピングしたひらがなが、そのまま **転がる主人公** になる 2D レースゲームの
プロトタイプ。
[Scribble Rider!](https://play.google.com/store/apps/details?id=com.tapped.drawrider) のように
「描いた形がタイヤになる」というアイデアを、「タイプしたひらがな **そのもの**
がコース上を転がっていく」に置き換えたものです。

物理エンジンは [Matter.js](https://brm.io/matter-js/) を使用し、凹ポリゴンの分解には
[poly-decomp](https://github.com/schteppe/poly-decomp.js) を併用しています。
ひらがなの形状は Canvas 上のラスタ描画から **Marching Squares** で輪郭ポリゴンを
抽出し、それを単体の剛体（compound body）にしてフィールドへ放り込みます。
**車体もタイヤもなく、ひらがな 1 個＝車両** というシンプルな構成です。

---

## 起動方法

依存はゲーム本体は CDN から読み込むためビルド不要です。  
ブラウザは `file://` だと一部の Canvas API（`getImageData`）が制限される場合があるので、
ローカルサーバ経由で開いてください。

### ひとりでプレイ

```bash
cd MojiRacing
npx serve . -p 8080
```

[http://localhost:8080](http://localhost:8080) を開き、「ひとりでプレイ」を選びます。

### 複数人プレイ（ローカル開発）

マルチプレイには [PartyKit](https://partykit.io/) の WebSocket サーバーが必要です。

```bash
npm install
npm run dev          # PartyKit（既定: localhost:1999）
# 別ターミナルで
npm run dev:web      # 静的ファイル（:8080）
```

1. [http://localhost:8080](http://localhost:8080) を開く  
2. ルームコードを決めて「ルームに参加」  
3. 友達に同じ URL（`?room=コード`）を共有  
4. 誰かが「レース開始」→ 3 秒カウントダウン後に同時スタート  

各プレイヤーは **自分の PC 上で物理シミュレーション** を行い、位置・文字だけを同期します（他プレイヤーは色付きのひらがなとして表示されます）。

## Vercel へのデプロイ

ゲーム本体（静的ファイル）とマルチサーバー（PartyKit）は **別々に** デプロイします。

### 1. PartyKit（マルチプレイ用）

```bash
npm install
npx partykit login   # 初回のみ
npm run deploy:party
```

表示されるホスト名（例: `mojiracing.yourname.partykit.dev`）を控えます。

### 2. Vercel（フロントエンド）

1. [Vercel](https://vercel.com/) でこのリポジトリをインポート  
2. **Environment Variables** に追加:  
   - `MOJI_PARTY_HOST` = `mojiracing.yourname.partykit.dev`（`https://` は付けない）  
3. デプロイ（`vercel.json` の `buildCommand` が `config.js` を自動生成します）

### 3. 遊び方

- デプロイ URL を開く → ルーム参加 → 「リンク」で `?room=XXXXX` 付き URL を共有  
- 全員が参加したら「レース開始」

## 遊び方

入力欄は **常にフォーカスが当たっている** ので、ページを開いたら何もしなくても
ひらがなをタイプするだけで、走行中の **ひらがな本体** がその場で別のひらがなに
入れ替わります。**位置・タイマー・速度・回転はそのまま** なので、走りながら
コブの直前で形を変えて飛距離を伸ばす、といった遊び方ができます。

1. ページを開く（既定: `あ` がコロコロ転がり始めます）
2. キーボードでひらがなをタイプ → そのまま走行を続けつつ「中の人」だけが変わる
3. ゴールフラッグまでの到達タイムを競ってください

### 操作

| キー | 動作 |
| ---- | ---- |
| `→` | アクセル（デフォルトの 1.5 倍まで） |
| `←` | ブレーキ・逆回転 |
| `↑` / `Space` | ジャンプ（少しだけ前回りスピンも乗ります） |
| ひらがなをタイプ | **走行中も「中の人」をホットスワップ**（タイマーリセットなし） |
| `リセット` ボタン | 最初からやり直し |

> 入力欄が外れているときに限り、`A`/`D` で加減速、`R` でリスタートも使えます。
> 通常は入力欄にフォーカスが当たっているので、これらのアルファベットキーは
> ローマ字入力（IME）に回されます。

## こだわっているところ

- **車もタイヤもない、ひらがな自身が転がる**: 車体（シャーシ）も別パーツの
  タイヤも持ちません。`getCharacterShape` が抽出した凹ポリゴンを 1 つの
  compound body として生成して、その剛体そのものを地形に放ります。
  `ぬ` のように突起の多い文字はガタガタと不規則に跳ね、`の` のように
  丸い文字はスーッと滑らかに転がります。
- **複数ストロークに対応**: `い` のように離れたストロークがある文字は、
  poly-decomp で凸パーツに分解した上で、単一階層の compound body として
  1 つの剛体に合体させています。
- **物理は本物**: 重心・慣性モーメントはすべて Matter.js が形状から自動計算。
  重心の偏った形は走り出しでよろけ、対称な形はまっすぐ走ります。
- **加速はトルク経由**: 角速度を直接設定するのではなく、剛体の上端あたりに
  水平力を与えてトルクを生成 → 地面とのフリクションで前進力に変換します。
  滑り・空転もそのまま現れます。
- **ホットスワップ**: 走行中にタイプすると、現在の位置・姿勢・速度・回転速度
  をそのまま新しいひらがな剛体へ引き継いで差し替えます。タイマーは
  止まらず、レース中に「いまだけ別の形」に切り替えられます。
- **固定タイムステップ**: 16.67ms 固定で物理ステップを進めるため、フレーム
  レートが揺れても挙動が安定します。
- **キーボード中心の UI**: 入力欄はオートフォーカス + IME composition 対応で、
  クリック操作なしでひらがなだけ叩けば中の人が切り替わってリスタート不要で
  走行が続きます。

## 地形と「ひらがなの形」の相性

コースは 4 種類の地形セクターで構成されています。ひらがなの形状から計算する
4 つの特徴量（`roundness` / `paddleScore` / `spikiness` / `aspect`）に応じて
それぞれの地形での走りやすさが変わるようバランスしてあります。

| 地形 | 物理特性 | 得意な形（例） | 苦手な形（例） |
| --- | --- | --- | --- |
| **平野**（緑） | 通常摩擦・小さなうねり | 丸い形（`の` `お` `ぬ` `め`） | 極端に偏った形 |
| **ゴツゴツ**（砂） | 高摩擦・細かい連続バンプ | 凹凸が多い形（`ま` `そ` `き` `さ`） | 真円に近い形 |
| **長い坂道**（深緑） | 大きなうねり | きれいに転がる丸い形（`の` `ろ` `お` `あ`） | ギザギザで弾む形 |
| **水上**（青） | 摩擦ほぼゼロ・水抵抗あり | 縦長＆突起のある形（`し` `へ` `く`）が水を蹴って前進 | 真円（ツルツル空転して進まない） |

### 物理的な仕組み

- **平野**: 通常の物理。剛体と地面の摩擦で素直に走ります。
- **ゴツゴツ**: 地面の摩擦を高めに（friction 1.15）。さらに剛体の
  `spikiness` に応じた追加トルクが効くため、`ま` のような多脚な形は
  バンプを引っかけてグイグイ進みます。逆に丸い形には「跳ねロス」を
  模した軽い減衰がかかります。
- **長い坂道**: 大きな波形の地形。`roundness` に応じた小さな転がり補正
  ボーナスが乗るので、`の` のような滑らかな形は転がり続けやすくなります。
- **水上**: 地面の摩擦をほぼ 0 に（friction 0.04）。普通のドライブ力では
  空転して進めません。代わりに、毎フレーム
  `ω × paddleScore × aspect` に比例する **パドル推力** を水平力として
  加え、ステップごとに 3.5% の水抵抗をかけています。半径の振れが大きく
  縦長な形（`し` `へ` `く`）が水を強く蹴って一番速く渡れます。

UI の左下には現在地形と、いま走っているひらがなの **相性（★1〜★3）** が
表示されます。坂で苦戦したらゴツゴツの直前で `ま` に履き替える、水際で
`し` に履き替える、といった「タイピングで戦略を切り替える」プレイが
楽しめます。

## ファイル構成

```
MojiRacing/
├── index.html       # UI レイアウト
├── style.css        # スタイル
├── game.js          # 物理 / 形状抽出 / レンダリング / ゲームロジック
├── multiplayer.js   # ルーム参加・WebSocket 同期
├── config.js        # PartyKit ホスト（Vercel ビルドで生成）
├── party/server.ts  # PartyKit ルームサーバー
├── vercel.json      # Vercel 設定
├── test-ms.mjs      # Marching Squares のロジック単体テスト（node で実行）
└── README.md
```

ロジックの正しさを確認するには:

```bash
node test-ms.mjs
```

`game.js` の主要なセクション:

| セクション | 役割 |
| -- | -- |
| `buildBinaryGrid` | ひらがなを Canvas に描画 → 2 値ピクセルグリッドへ |
| `marchingSquares` | 2 値画像から閉じた輪郭ポリゴン群を抽出 |
| `simplifyDP` | Douglas–Peucker で頂点を削減 |
| `getCharacterShape` | 上記をまとめてゲーム用の正規化されたポリゴン群を返す |
| `computeShapeStats` | 形状から `roundness` / `paddleScore` / `spikiness` / `aspect` を算出 |
| `createCharacterBody` | 各輪郭から Matter.js の剛体を生成し、複数輪郭は compound body に合体 |
| `createVehicle` / `swapShape` | ひらがな剛体 1 個を生成 / 走行中に位置・速度を維持したままホットスワップ |
| `SECTOR_PLAN` / `surfaceY` / `terrainMaterialFor` | 地形を `plain` / `bumpy` / `slope` / `water` のセクターに分け、起伏式と摩擦・反発を切り替える |
| `createTerrain` | セクター別の物性で回転矩形セグメントを生成 |
| `applyControls` | キー入力 → ひらがな剛体へのトルク・ジャンプ力 |
| `applyTerrainEffects` | 地形ごとの追加力（水上のパドル推力・水抵抗、ゴツゴツのグリップ補正、坂の転がり補正）を毎ステップ適用 |
| `loop` | 固定ステップ物理 + 描画 |

## 既知の制限 / 今後の改善案

- 複雑すぎる文字（極端に細い線が多いものなど）では、ポリゴン分解が
  失敗して剛体が小さくなったり丸くなることがあります。フォントウェイトの
  調整や輪郭の太め化（モルフォロジー膨張）で改善余地があります。
- 現在は外輪のみ採用し、文字の「穴」（`ぬ` の中の閉領域など）は無視しています。
- ステージは 1 種類だけです。レベルセレクトや障害物追加は今後の拡張予定。
- カタカナ・漢字・濁点付きひらがなへの拡張は、入力バリデーションを変えるだけで
  すぐ対応できます（`isHiragana` を緩めるだけ）。
